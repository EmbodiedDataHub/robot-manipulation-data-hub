## 一句话总结

UMI 用 **手持夹爪 + GoPro** 在任意环境中采集人类演示，通过精心设计的 **硬件接口**（鱼眼、侧镜、IMU-SLAM、连续夹爪宽度）和 **策略接口**（延迟对齐、相对轨迹动作、Diffusion Policy），把野外人类演示 **零样本迁移** 到可部署的机器人策略，支持动态、双臂、精细、长时域操作，并在未见环境/物体上达到约 **70%** 成功率。

---

## 1. 核心问题

机器人模仿学习的数据从哪里来？现有两条路都有明显短板：

| 路线 | 优点 | 问题 |
|------|------|------|
| **遥操作机器人** | 动作空间与机器人一致，可直接迁移 | 需要真实机器人、专家操作员、高 setup 成本；数据与具体 embodiment 绑定 |
| **野外人类视频** | 视觉多样、场景开放 | 缺少显式动作；人-机 embodiment gap 大；观测空间不一致 |

**手持夹爪**（如 Dobb-E、类似工作）是折中方案，但此前工作普遍只能做 **准静态 pick-and-place**，难以迁移 **动态、双臂、精细** 动作。

论文指出四个关键障碍：

1. **视觉上下文不足**：腕部相机离物体太近，遮挡严重
2. **动作不精确**：单目 SfM 有尺度歧义、运动模糊、纹理不足
3. **延迟不一致**：采集时观测/动作同步，部署时相机/推理/执行各有延迟
4. **策略表示不足**：简单 MLP + 回归难以拟合人类数据的多模态分布

UMI 从 **演示接口设计** 和 **策略接口设计** 两侧同时解决这些问题。

## 2. 系统概览

```
人类演示（任意环境）
    │
    ▼
┌─────────────────────────────────────┐
│  UMI 手持夹爪                        │
│  GoPro(鱼眼+IMU) + 平行夹爪 + 侧镜   │
│  → 单文件 MP4 记录全部信息           │
└─────────────────────────────────────┘
    │
    ▼ SLAM 恢复 6DoF 轨迹 + 夹爪宽度
    │
    ▼ 运动学过滤（按目标机器人筛选可行轨迹）
    │
    ▼ Diffusion Policy 训练
    │
    ▼ 延迟对齐 + 相对轨迹动作
    │
    ▼
机器人部署（UR5 / Franka 等，硬件无关）
```

**设计目标**：

- **Portable**：接近零 setup，可带到任何环境
- **Capable**：超越 pick-and-place 的复杂操作
- **Sufficient**：信息足够学策略，且 embodiment 信息最小化
- **Reproducible**：低成本可复现（夹爪 BOM ~$73，GoPro 及配件 ~$298）

## 3. 演示接口设计（Hardware Interface）

硬件：**3D 打印平行夹爪 + 软指 + GoPro（唯一传感器）**，双手任务加第二个夹爪即可。总重 **780g**，尺寸约 310×175×210 mm，开合行程 **80mm**。

### HD1. 腕部相机作为唯一观测

- 人类演示与机器人部署时，相机相对夹爪/手指位置 **完全一致**
- 无需外置相机、无需 camera-robot-world 标定
- 移动相机带来类似 random crop 的数据增强，策略对背景干扰更鲁棒

### HD2. 155° 鱼眼镜头

- 提供足够 **视觉上下文**，减少物体出画
- **直接使用原始鱼眼图像**，不做去畸变——中心区域保分辨率，边缘压缩
- 对比：把 155° 鱼眼 rect 成 pinhole 会严重拉伸边缘、压缩中心，策略性能从 **100% → 55%**

### HD3. 侧镜实现隐式立体

- 在鱼眼视野边缘放置 **物理侧镜**，等价于额外虚拟相机
- 对镜中区域做 **数字翻转 + 左右交换** 后送入策略，成功率 **100%**（无镜 90%，有镜但不翻转 85%）
- 解决单目深度感知不足的问题

### HD4. IMU-aware SLAM 动作跟踪

- 利用 GoPro 内置 **IMU**（加速度计+陀螺仪，写入 mp4）
- 基于 **ORB-SLAM3** 的视觉-惯性联合优化
- 优势：
  - 恢复 **真实 metric 尺度**（解决单目 SfM 尺度歧义）
  - 运动模糊/纹理缺失时短时仍可跟踪
  - 支持 **动态动作**（如抛掷）和 **双臂相对位姿**

**数据采集流程**：先拍一段视频 **建图**，后续演示 **重定位到同一 map**，保证双臂共享坐标系。

### HD5. 连续夹爪宽度控制

- 不用二值开/关，而是 **连续宽度**（ArUco 标记跟踪）
- 软指 + 连续宽度 ≈ 隐式记录/控制 **抓取力**
- 对抛掷等需要精确 **释放时机** 的任务至关重要

### HD6. 基于运动学的数据过滤

- SLAM 恢复绝对 EE 位姿后，按目标机器人 **运动学/动力学可行性** 过滤无效轨迹
- 采集阶段 robot-agnostic，训练阶段 embodiment-specific

## 4. 策略接口设计（Policy Interface）

策略：**Diffusion Policy**（论文称 ACT 等也可替换）。

**输入**：同步的观测序列（RGB 图像 + 相对 EE 位姿 + 夹爪宽度）

**输出**：期望的相对 EE 位姿序列 + 夹爪宽度

### PD1. 推理时延迟对齐（Latency Matching）

采集时 GoPro 内视频/IMU/夹爪宽度 **零延迟同步**；部署时各流延迟不同（相机 30Hz、机械臂 125Hz、夹爪 10Hz 等）。

**观测侧（PD1.1）**：
- 分别测量各流延迟
- 以 **最高延迟流（通常是相机）** 为基准
- 用图像时间戳对 EE 位姿和夹爪宽度做 **线性插值**
- 双臂：软同步，最近邻帧对齐（最大偏差 1/60 秒）

**执行侧（PD1.2）**：
- 提前发送指令以补偿 **执行延迟**
- 丢弃因观测延迟 + 推理延迟 + 执行延迟而 **过时的动作步**

**效果**：动态抛掷任务中，关闭延迟对齐成功率从 **87.5% → 57.5%**

### PD2. 相对 EE 轨迹表示

所有 EE 位姿表示为 **相对于当前 EE 位姿的 SE(3) 变换序列**。

| 动作空间 | 杯排列成功率 | 主要问题 |
|----------|-------------|----------|
| **相对轨迹（UMI）** | **100%** | — |
| Delta（逐步相对上一步） | 80% | 误差累积 |
| Absolute（机器人基坐标） | 25% | SLAM-机器人标定误差，动作偏置 |

**好处**：
- 对 SLAM 跟踪误差、相机位移更鲁棒
- **无需标定**：移动机器人基座不影响任务（只要够得着）
- 适用于移动操作

**双臂额外项（PD2.3）**：提供 **两夹爪间相对位姿** 作为本体感知。去掉后双臂叠衣服成功率 **70% → 30%**。

## 5. 实验任务与结果

### 5.1 能力实验（窄域，同环境不同初始状态）

| 任务 | 难点 | 演示数 | 成功率 | 备注 |
|------|------|--------|--------|------|
| **杯排列** | 抓取+非抓取（推转杯柄）、多模态（顺/逆时针） | 305 | **20/20** | Franka 跨 embodiment **18/20** |
| **动态抛掷** | 6 个 YCB 物体分拣到圆/方 bin，需抛掷 | 280 | **87.5%** (105/120) | 无延迟对齐仅 57.5% |
| **双臂叠衣服** | 可变形物体、双臂 tight 协调 | 250 | **70%** (14/20) | 无 inter-gripper pose 仅 30% |
| **洗碗** | 7 步长时域：开水龙头、洗碟、擦番茄酱、关水龙头等 | 258 | **70%** (14/20) | 需 CLIP ViT；ResNet 0/10 |

**杯排列消融要点**：
- 无鱼眼：55%
- 无侧镜数字翻转：85% → 有翻转：100%
- 无延迟对齐（抛掷）：57.5%

**洗碗额外鲁棒性**（Fig. 10）：
- 移动机器人基座 ✓
- 新物体（松果、盒子）✓
- 不同光照 ✓
- 洗涤过程中/后追加不同酱料 ✓

### 5.2 野外泛化实验（Cup Arrangement 扩展）

- **12 人时**，3 名演示者，**30 个物理位置**（家、办公室、餐厅、户外）
- **1400 条**演示，**15 种**不同杯子（颜色/形状/材质）
- 视觉编码器：CLIP ViT-L/14

**测试环境**（训练时未见）：
- **Cafe table**：户外繁忙咖啡厅，行人干扰
- **Water fountain**：黑色立方体喷泉，表面持续流水（OOD 程度极高）

| 测试集 | 成功率 |
|--------|--------|
| 训练见过的杯子 | 28/40 = **70%** |
| 未见杯子（深蓝 OOD 色 + 棕色环纹新纹理） | 15/20 = **75%** |
| **合计** | **43/60 = 71.7%** |

**对照**：仅用窄域实验室数据 + 同样 ViT backbone → 未见环境 **0%**（机器人甚至不朝杯子移动）。

**结论**：仅靠预训练视觉 backbone 不够，**野外多样化数据** 才是泛化关键。

## 6. 数据采集效率与 SLAM 精度

### 吞吐量（15 分钟对比）

| 方法 | 杯排列 (CPH) | 动态抛掷 |
|------|-------------|----------|
| 人手 | 231 | 237 |
| **UMI** | **111** (~人手 48%) | **149** (~人手 64%) |
| SpaceMouse 遥操作 | 35 | **0**（15 分钟内无成功演示） |

UMI 比传统遥操作 **>3× 更快**，且能完成遥操作无法采集的动态任务。

### SLAM 精度（MoCap 基准，7 单臂 + 7 双臂任务）

| 指标 | 位置 | 旋转 |
|------|------|------|
| 单夹爪 ATE | **6.1 mm** | **3.5°** |
| 双臂 inter-gripper RPE | **10.1 mm** | **0.8°** |

## 8. 局限与未来方向

1. **运动学过滤**：采集时不知下游机器人约束，只能事后过滤；未来可做 embodiment-aware 策略学习
2. **SLAM 依赖纹理**：白墙等低纹理环境易失败；可考虑第三人称相机 + 夹爪 fiducial
3. **采集效率仍低于人手**：夹爪较重（780g）、DoF 少于人手；可改进 ergonomics 或探索灵巧手直接迁移
4. **硬件限制**：平行夹爪，非全灵巧操作

---

## 9. 关键 Takeaways

1. **中间路线可行**：手持夹爪 + 正确接口设计，可以在 **无机器人** 的情况下采集 **可部署** 的高质量操作数据
2. **接口设计 > 单纯堆数据**：鱼眼、侧镜、延迟对齐、相对轨迹每一项消融都带来显著性能差异
3. **观测对齐是核心**：腕部相机在人类演示和机器人部署间保持一致，极大缩小 embodiment gap
4. **Diffusion Policy 拟合多模态人类演示**：如杯柄旋转的顺/逆时针两种 equally valid 解
5. **野外数据的不可替代性**：预训练 ViT + 窄域数据 ≠ 泛化；需要 **in-the-wild 多样化演示**
6. **开源**：硬件与软件均开源 → https://umi-gripper.github.io

# 数据格式术语对照与解析入门

> **问题**：机器人里的「格式」和目标检测里的 COCO/YOLO 是一回事吗？  
> **答案**：**是同一类概念（数据约定 + 序列化 + 加载工具链）**，但**数据结构不同**——检测是「单张图 + 静态标注」，机器人是「时间序列 episode + 多模态观测 + 动作」。

---

## 一、和 COCO / YOLO 的对照

| 层次 | 目标检测 (COCO/YOLO) | 机器人操作 (HDF5/LeRobot/RLDS) |
|------|------------------------|--------------------------------|
| **语义 schema** | 一张图对应若干 bbox + category_id | 一条 episode 对应 T 步 (observation, action, language) |
| **序列化** | JSON (COCO) / 文本行 (YOLO) / XML | HDF5 / Parquet+MP4 / TFRecord |
| **加载库** | `pycocotools`, `ultralytics` | `h5py`, `lerobot`, `tensorflow_datasets` |
| **训练单位** | 单张 image | 单步 step 或整段 episode（常带 action chunk） |
| **多模态** | 通常只有 RGB | RGB × 多相机 + 关节状态 + 语言 + 可选深度/力 |

可以记成：

```text
COCO  = 图像检测的「接口标准」
RLDS  = 跨机器人预训练的「接口标准」
LeRobot = HF 生态微调的「接口标准」
HDF5  = IL 传统项目（ACT/robomimic/LIBERO）的「事实存储」
```

**不是文件后缀这么简单**：YOLO 既有 `.txt` 标注也有目录约定；机器人同理——LeRobot 是一个**目录结构**（`meta/` + `data/` + `videos/`），不只是某一种扩展名。

## 二、三种格式的「长什么样」

### 2.1 HDF5（ALOHA / ACT 风格）

**一个 episode = 一个 `.hdf5` 文件**：

```text
episode_0.hdf5
├── @attrs: sim=True
├── observations/
│   ├── images/
│   │   ├── cam_high          [T, H, W, 3] uint8
│   │   ├── cam_left_wrist
│   │   └── cam_right_wrist
│   ├── qpos                  [T, 14] float
│   └── qvel                  [T, 14] float
└── action                      [T, 14] float
```

**robomimic / LIBERO** 则是「一个文件里多个 demo」：

```text
dataset.hdf5
└── data/
    ├── demo_0/
    │   ├── actions           [T, D]
    │   └── obs/
    │       ├── agentview_image
    │       └── ...
    └── demo_1/
        └── ...
```

**本仓库解析代码**：

- 读取：`code/act/visualize_episodes.py` → `load_hdf5()`
- 训练采样：`code/act/utils.py` → `EpisodicDataset`
- Demo：`code/robot-dataset-demos/scripts/inspect_hdf5.py`

```python
import h5py
with h5py.File("episode_0.hdf5", "r") as f:
    action = f["action"][t]           # 第 t 步动作
    qpos = f["observations/qpos"][t]    # 第 t 步关节
    img = f["observations/images/cam_high"][t]  # HWC uint8
```

### 2.2 LeRobot（Hugging Face 生态）

**一个数据集 = 一个目录**：

```text
my_dataset/
├── meta/
│   └── info.json          # schema：features、fps、路径模板
├── data/
│   └── chunk-000/
│       └── episode_000000.parquet   # 每行 = 一帧（action, state, index...）
└── videos/
    └── chunk-000/
        └── observation.images.top/
            └── episode_000000.mp4   # 图像单独存视频，parquet 只存索引/元数据
```

`info.json` 相当于 LeRobot 的「COCO categories + 字段定义」。

**官方库**：[huggingface/lerobot](https://github.com/huggingface/lerobot)

```python
from lerobot.common.datasets.lerobot_dataset import LeRobotDataset
ds = LeRobotDataset("lerobot/aloha_sim_transfer_cube_human")
sample = ds[0]  # 返回一帧的所有字段
```

**本仓库 Demo**（不装 lerobot 也能看结构）：

```bash
cd code/robot-dataset-demos
python scripts/inspect_lerobot.py samples/lerobot_minimal
# 真实 HF 样例（已下载）：
python scripts/inspect_lerobot.py samples/lerobot_aloha_sim_hf
```

### 2.3 RLDS（OXE / DROID / Bridge）

**逻辑 schema**（序列化在磁盘上是 TFRecord）：

```text
Episode
 ├── episode_metadata: {robot_type, ...}
 └── steps[]: 长度 T 的序列
      ├── observation: {image_*, state, ...}
      ├── action: float32[D]
      ├── language_instruction: string
      └── is_first / is_last / reward ...
```

相当于「带时间维的 COCO」——每一步是一张「逻辑帧」，但字段比检测标注多得多。

**官方工具**：

- [open_x_embodiment](https://github.com/google-deepmind/open_x_embodiment)
- `tensorflow_datasets` → `tfds.load("bridge_dataset", ...)`
- **零安装体验**：[OXE Colab](https://colab.research.google.com/github/google-deepmind/open_x_embodiment/blob/main/colabs/Open_X_Embodiment_Datasets.ipynb)

```python
import tensorflow_datasets as tfds
ds = tfds.load("droid_100", split="train", data_dir="~/tensorflow_datasets")
for episode in ds.take(1):
    for step in episode["steps"].take(1):
        print(step["action"].numpy())
        print(step["observation"]["exterior_image_1_left"].shape)
```

**本仓库 Demo**：`code/robot-dataset-demos/scripts/inspect_rlds.py`（需本地已下载 TFDS 数据）

## 三、和检测 pipeline 的类比图

```text
目标检测                         机器人模仿学习
─────────                        ──────────────
原始图 jpg/png                   原始 bag/MCAP 或 相机+关节流
    ↓                                ↓
标注工具 → COCO JSON             遥操作录制 → episode 边界
    ↓                                ↓
COCO API / YOLO loader           h5py / LeRobot / TFDS loader
    ↓                                ↓
batch: image + bbox label        batch: image + action (+ language)
```

## 一句话

**仿真 / 合成 Data Engine** 用 **MuJoCo/Isaac 等仿真器 + 脚本/RL/LLM/MimicGen 扩增**，从少量种子 demo 自动生成 **万级轨迹**——本仓库 **范式 E**，scale 极大但面临 **sim2real gap**，适合 **Tier 2 预训练 bootstrap**，需 domain randomization 和真机校准。

---

## 1. 为什么需要仿真数据

| 优势 | 局限 |
|------|------|
| 自动 success 标签 | 渲染/物理 simplified |
| 200 种子 → 50K 轨迹 | 接触/摩擦 modeling 误差 |
| 无机器人硬件成本 | 观测 domain gap |
| LLM 自动生成任务 | failure mode 覆盖仍不足 |

---

## 2. 本仓库核心论文

### 2.1 MimicGen ⭐

| 项 | 内容 |
|----|------|
| 输入 | **~200** 人类 teleop 种子（仿真或 real） |
| 方法 | 物体中心 **子任务分解** + 空间变换 + 仿真拼接 |
| 输出 | **~50,000** 成功轨迹 |
| Action | 与 robosuite 一致，通常 **EEF delta** |

**管线**：

```text
种子轨迹 → 切分子段 [接近, 抓取, 移动, 放置]
        → 随机化物体位姿/纹理
        → 仿真中重放子段并拼接
        → 自动过滤失败 rollout
        → 导出 (observation, action) 对
```

### 2.2 其他（综述 / 索引）

| 工作 | 特点 |
|------|------|
| **RoboGen** | LLM 生成任务+奖励 → 自动 RL |
| **RoboCasa** | 厨房场景大规模家居 |
| **SynGrasp-1B** | 十亿级合成抓取 |
| **LIBERO** | 基准+仿真环境（评估为主，也可产数据） |

## 3. 仿真数据的「教师」类型

| 教师 | 原理 | 数据质量 |
|------|------|----------|
| 脚本专家 | 抓取点采样 + IK | 可行但多样性低 |
| 运动规划 | RRT/PRM | 无碰撞，可能不 natural |
| RL 策略 | 仿真内训练至收敛 | 取决于 reward 设计 |
| MimicGen | 人类 demo 空间变换 | **最接近 human demo** |

## 4. 前处理（→ Canonical IR）

```yaml
source: sim_mimicgen
robot_type: "franka_sim" / "widowx_sim"
success: true                    # 仿真自动判定
metadata:
  domain_rand: {light, friction, texture}
  seed_episode_id: "..."

steps:
  - observations:
      images: {exterior_1: sim_render}
      proprio: {joint_pos, eef_pose}
    action:
      eef_delta: [...]           # 与 deploy 一致！
```

**关键规则**：

1. **Sim action 空间 = deploy action 空间**（通常 EEF delta）
2. **Domain randomization** 必开（光照、摩擦、质量、相机 pose 噪声）
3. 混合真机数据时加 **embodiment_id** + **较低 mixing weight** 起步
4. 2026 实践：VLM critic 过滤「物理不合理」轨迹

---

## 5. 与真机数据混合

```text
Data Pyramid:
  Tier 2: MimicGen 50K (sim) + OXE 子集
  Tier 3: 目标 robot 100 ep teleop（决定 sim2real 上限）

负迁移风险:
  纯 sim 预训练 → 真机 finetune 可能仍需 hundreds demos
  建议: sim:real 混合比例 ablation（从 9:1 → 1:1 逐步试）
```

## 8. 选型

| 场景 | 用仿真 |
|------|--------|
| 无真机 bootstrap | ✅ |
| 需要百万级 diversity | ✅ MimicGen/LLM gen |
| 精细接触 / 柔性物体 | ⚠️ 谨慎，sim gap 大 |
| 最终部署成功率 | 必须补 Tier 3 teleop |

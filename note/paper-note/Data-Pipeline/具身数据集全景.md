# 具身智能机器人数据集全景

> **目标**：建立对主流机器人操作数据集的**全局认知**——规模、用途、存储格式、下载方式。  
> **关联**：[Open X 与 RLDS 格式](./Open-X与RLDS格式.md) · [VLA 数据综述](../VLA-Datasets-Benchmarks-Data-Engines.md) · [资源索引](../../resources/links/资源索引.md)

---

## 一句话

当前主流数据集可按 **数据来源**（真机 / 手持 / 仿真 / ego 视频）、**用途**（预训练 / 微调 / Benchmark）、**格式**（RLDS / LeRobot / HDF5）三轴理解；无论来源如何，训练接口都收敛为 `(观测 ot, 语言 l, 动作 at)`。

## 一、全局分类

| 维度 | 典型划分 |
|------|----------|
| **数据来源** | 真机遥操作 / 手持野外(UMI) / 仿真 / 人类 ego 视频 |
| **用途** | 跨 embodiment 预训练 / 单平台微调 / Benchmark 评测 |
| **存储格式** | RLDS · LeRobot · HDF5 · MCAP/ROS bag |

## 二、存储格式速查（最重要）

理解所有数据集的基础。详见 [Open X 与 RLDS 格式](./Open-X与RLDS格式.md) · [术语对照与解析入门](./数据格式术语对照与解析入门.md)（含 COCO/YOLO 类比与 Demo 子仓库 [`code/robot-dataset-demos`](../../../code/robot-dataset-demos)）。

### 2.1 RLDS / TFRecord（跨机构预训练事实标准）

- **生态**：TensorFlow / JAX；OXE、Octo、OpenVLA、π0 全量 DROID 训练
- **结构**：`Episode → steps[]`，每步含 `observation` / `action` / `language_instruction`
- **特点**：异构 action dim、多相机都能容纳；适合大规模混合训练
- **工具**：[google-deepmind/open_x_embodiment](https://github.com/google-deepmind/open_x_embodiment)

```text
Episode
 ├── episode_metadata (robot_type, ...)
 └── steps[]
      ├── observation: {image_*, state, ...}
      ├── action: float32[D_action]
      └── language_instruction: string
```

### 2.2 LeRobot（Hugging Face 生态，2025–2026 快速普及）

- **存储**：`Parquet`（状态/动作/元数据）+ `MP4`（视频）
- **生态**：PyTorch、`lerobot-train`、SmolVLA、OpenPI 微调
- **特点**：对中小规模真机数据友好；大 TB 级数据转换仍在推进
- **工具**：[huggingface/lerobot](https://github.com/huggingface/lerobot)

### 2.3 HDF5（传统 IL 主流）

- **生态**：ACT、robomimic、LIBERO、ALOHA 原始采集
- **结构**：每个 episode 一个 `.hdf5`，内含图像数组或视频引用 + 关节/动作
- **特点**：成熟、工具多；跨项目互通性不如 RLDS/LeRobot

### 2.4 MCAP / ROS2 bag（采集原始格式）

- **用途**：真机录制中间态，QA 后导出为 LeRobot 或 RLDS
- **不直接用于** 大多数 VLA 训练，但是数据管线起点

### 2.5 格式选型建议

| 场景 | 推荐格式 |
|------|----------|
| 贡献 OXE / 大规模预训练 | RLDS |
| 自己采数据、HF 微调 | LeRobot |
| 跑 ACT / robomimic / LIBERO | HDF5 |
| 真机录制 | MCAP → 再转换 |

**双轨策略**（见 [总方案](../../数采到VLA训练-数据管线整体方案.md)）：

```text
Canonical IR ─┬─→ RLDS exporter（OXE 贡献）
              └─→ LeRobot exporter（HF 微调）
```

## 三、主要数据集分层总表

### Tier 0：聚合层（预训练入口）

| 数据集 | 规模 | 机器人 | 格式 | 核心用途 | 下载 |
|--------|------|--------|------|----------|------|
| **Open X-Embodiment (OXE)** | 1M+ episodes，22 种机器人，60 子集 | 多机构混合 | RLDS | 跨 embodiment 预训练 | [GitHub](https://github.com/google-deepmind/open_x_embodiment) · [官网](https://robotics-transformer-x.github.io/) · [子集元数据表](https://docs.google.com/spreadsheets/d/1rPBD77tk60AEIGZrGSODwyyzs5FgCU9Uz3h-3_t2A9g/edit#gid=0) |
| **HF 镜像（非官方）** | ~1.13 TB | 同上 | RLDS 转 HF | 不用 gsutil 时 | [jxu124/OpenX-Embodiment](https://huggingface.co/datasets/jxu124/OpenX-Embodiment) |
| **LeRobot 转换版** | 部分子集 | 同上 | LeRobot v2/v3 | PyTorch 微调 | [openx_dataset_lerobot_v2.0](https://huggingface.co/spaces/IPEC-COMMUNITY/openx_dataset_lerobot_v2.0) · [openx2lerobot](https://github.com/Tavish9/openx2lerobot) |

**下载命令（官方 GCS）**：

```bash
# 安装 gsutil 后，按子集名下载（注意 dataset 名不要多下划线）
gsutil -m cp -r gs://gresearch/robotics/{dataset_name} ~/tensorflow_datasets/
# 例：RT-1 子集
gsutil -m cp -r gs://gresearch/robotics/fractal20220817_data ~/tensorflow_datasets/
```

### Tier 1：大规模真机单数据集（微调 / 混训核心）

| 数据集 | 规模 | 平台 | Action 空间 | 格式 | 特点 | 下载 |
|--------|------|------|-------------|------|------|------|
| **RT-1** | ~130K episodes | Everyday Robots 移动臂 | Delta EEF + gripper (7D) | RLDS（OXE 子集 `fractal20220817_data`） | 多任务、车队级；VLA 谱系起点 | 通过 OXE 下载 |

**DROID 下载**：

```bash
# RLDS 完整版 1.7TB（推荐 v1.0.1，含完整语言标注）
gsutil -m cp -r gs://gresearch/robotics/droid/1.0.1 <your_path>

# 小样测试（~2GB）
gsutil -m cp -r gs://gresearch/robotics/droid_100 <your_path>

# 原始全分辨率（8.7TB）
gsutil -m cp -r gs://gresearch/robotics/droid_raw <your_path>
```

**LeRobot 版 DROID**（不想自己转）：[cadene/droid_1.0.1](https://huggingface.co/datasets/cadene/droid_1.0.1)

## 四、Action 空间差异（混训时必须注意）

| 数据集 | Action 类型 | 维度 | 备注 |
|--------|------------|------|------|
| RT-1 / Bridge | Delta EEF + gripper | 7 | OpenVLA 默认归一化目标 |
| DROID | **Absolute** EEF + gripper | 7 | 与 Bridge 不同，混训需转换 |
| ALOHA | Joint positions | 14 | 双臂各 6 关节 + 2 夹爪 |
| UMI | Relative EEF | 7 | 与绝对定位解耦 |
| RDT | Unified bimanual | 128 | pad/truncate 到统一空间 |

OXE 能共存不同 action dim，是因为训练时用 **per-dataset normalization + embodiment embedding**；但 2026 共识是：**盲目混合可能负迁移**，需要 Data Pyramid 和 curated mixing。详见 [VLA 训练与数据全貌 §9](../../VLA训练与数据全貌-深度版.md)。

## 七、一条训练样本的 anatomy

无论数据集来源，Episode / Step 层级一致：

```text
Dataset
 └── Episode (一次任务尝试，如「把红杯子放到左边」)
      ├── metadata: {robot_type, lab, date, success, instruction_text, ...}
      └── Step × T  (通常 50–500 步，20–60 秒)
           ├── observation
           │    ├── image_exterior: (H, W, 3) uint8
           │    ├── image_wrist: (H, W, 3) uint8      # 可选
           │    ├── proprio: (D,) float                 # 关节角或 EEF pose
           │    └── language: str                       # 通常 episode 级
           └── action
                └── (D_action,) float                  # 下一步或 chunk 目标
```

详见 [VLA 训练与数据全貌 §4](../../VLA训练与数据全貌-深度版.md)。

## 九、2026 趋势

1. **格式收敛**：RLDS（预训练）+ LeRobot（微调）双轨并行，互转工具在完善
2. **质 > 量**：5% curated coreset 可恢复 85–90% 全量性能；负迁移是真实问题
3. **语言标注变重要**：DROID v1.0.1 补全了 ~75K episodes 的语言标注
4. **仿真数据回归**：RoboCasa365、MimicGen 等用于 bootstrap，但 sim-to-real gap 仍在

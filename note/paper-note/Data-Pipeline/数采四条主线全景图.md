# 数采四条主线全景图（ALOHA · UMI · Ego · VR）

> **范围**：范式 A（Teleop）、B（手持 UMI）、C（VR 人形）、D（Ego 视频）的采集方法、演进树、对比与选型。  
> **互补**：[Data Pipeline 概述](./概述.md) · [数采到 VLA 训练总方案](../../数采到VLA训练-数据管线整体方案.md) · [具身数据集全景](./具身数据集全景.md)

---

## 一句话

四条数采主线 = **真机高精度（ALOHA）→ 无真机手持（UMI）→ 人形沉浸（VR）→ 纯视频规模化（Ego）**；fidelity 递减、scale 递增；升级版多在硬件工程化、追踪可靠性、移动/灵巧扩展、数据规模四方向；训练上 **Ego 做 prior，UMI/Teleop 做可执行 action，Tier 3 robot-aligned 数据决定部署成功率**。

---

## 一、总框架

所有数采最终收敛为训练接口：

```text
(o_t, ℓ, a_t)  =  观测（图像 + 状态）+ 语言 + 动作
```

差别只在 **怎么采、action 从哪来、fidelity vs scale**：

```text
                    数采全景（Fidelity ↑  vs  Scale ↑）

  Fidelity 高、Scale 小                    Scale 大、Fidelity 低
  ─────────────────────────────────────────────────────────────
  A. Teleop          B. 手持 UMI         D. Ego 视频
  (ALOHA/DROID)      (无真机)             (普通人类视频)
        │                  │                    │
        │            C. VR Teleop               │
        │            (人形 whole-body)          │
        └──────────────────┴────────────────────┘
                           │
                    Canonical IR → LeRobot / RLDS → VLA 训练
```

### Data Pyramid（2026 共识）

```text
        Tier 3 · 精调：Teleop / UMI robot-aligned（决定最终成功率）
              ↑
        Tier 2 · 预训练：OXE / UMI 万小时 / 多源混训
              ↑
        Tier 1 · 先验：Ego 视频 / Web VLM（语义 breadth，不能单独部署）
```

## 二、范式 A · 实验室 Teleop（ALOHA 生态）

### 2.1 核心采集方法

```text
人操作 Leader（主端）──关节映射──→ Follower（从端）真机执行
         │                              │
         └──────── 同步录制 ─────────────┘
              多相机 RGB + qpos + action + 时间戳
```

| 项 | 内容 |
|----|------|
| **采集时要机器人** | ✅ Follower 真机 |
| **原始数据** | 相机流 + 关节编码器（精确） |
| **Action** | **14D 绝对关节角**（双臂） |
| **相机** | 固定俯视 `cam_high` + 双腕相机 |
| **存储** | HDF5 / LeRobot |
| **Fidelity** | ⭐⭐⭐⭐⭐ |
| **Scale** | 小（贵、慢） |

### 2.2 演进树

```text
ALOHA (2023)          <$20k 双臂 ViperX+WidowX + ACT
    │
    ├── ALOHA 2 (2024)        硬件车队化：夹爪/RealSense D405/重力补偿/MuJoCo
    │
    ├── Mobile ALOHA (2024)     +移动底座，16D action，静态+移动 co-train
    │       └── TidyBot++       开源全向家用底座 + 手机 teleop + 任意臂
    │
    ├── GELLO (2023)            3D 打印外骨骼 leader，<$30，适配 Franka/UR5/xArm
    │
    ├── AirExo                  外骨骼野外采 whole-arm 数据 + 少量 teleop
    │
    └── Giving Robots a Hand    少量 robot demo + 人类腕部视频增强泛化
```

### 2.3 相关路线（同范式 A）

| 论文 | 与 ALOHA 关系 |
|------|--------------|
| **DROID** | 单臂桌面 fleet，7D EEF，in-the-wild teleop |
| **RT-1** | 大规模车队 teleop，OXE 重要子集 |

## 三、范式 B · 手持野外（UMI 生态）

### 3.1 核心采集方法

```text
手持夹爪 + GoPro（腕部相机）
    ↓
录 MP4 + IMU
    ↓
离线：SLAM / T265 → 6D 位姿 + 夹爪宽度
    ↓
转 relative EEF 轨迹 + latency matching
    ↓
训练数据（采集阶段无需真机）
```

| 项 | 内容 |
|----|------|
| **采集时要机器人** | ❌ |
| **原始数据** | GoPro MP4（+ IMU） |
| **Action 来源** | **SLAM / 追踪** 反推 EEF |
| **相机** | **仅腕部**第一人称（与部署对齐） |
| **Action** | **7D 相对 EEF** + 夹爪宽（双臂 + 夹爪间位姿） |
| **Fidelity** | ⭐⭐⭐⭐ |
| **Scale** | 中→大 |

### 3.2 演进树

```text
UMI (2024)            GoPro + ORB-SLAM3 + 平行夹爪，野外无真机
    │
    ├── FastUMI         T265 替 SLAM；硬件模块化；万条开源数据集
    │
    ├── RDT2 UMI        红外追踪替 SLAM；连杆夹爪；10K+ 小时 fleet
    │
    ├── DexUMI          灵巧手：外骨骼 + inpainting（多指，非平行夹爪）
    │
    └── DexWild         纯人手野外 dexterous，更极端 scale
```

**笔记**：[UMI](../UMI-Universal-Manipulation-Interface.md) · [RDT §9](../RDT-Foundation-Models.md#part-b-rdt2)

## 四、范式 C · VR / 人形 Teleop

### 4.1 核心采集方法

```text
操作员戴 VR（Vision Pro / Quest / PICO）
    ↓
捕获 头/手/腕 6D pose
    ↓
IK + motion retargeting → 机器人关节角
    ↓
同步录制：主动立体头显图像 + whole-body joint
```

| 项 | 内容 |
|----|------|
| **采集时要机器人** | ✅ 人形/全身机器人现场 |
| **原始数据** | VR 追踪 pose + 机器人 proprio + 头戴相机 |
| **Action** | **whole-body 高维 joint**（臂+手+有时底座） |
| **相机** | **主动立体头显**（随头动，非固定第三人称） |
| **与 ALOHA 差异** | 关节 **IK 映射**（非同构 copy），更通用、可远程 |
| **Fidelity** | ⭐⭐⭐⭐ |
| **Scale** | 中 |

### 4.2 演进树

```text
Open-TeleVision (OTV)     Apple Vision Pro + 主动立体相机
    │                     Unitree H1 / Fourier GR-1
    │                     远程 teleop（Boston↔San Diego）
    │
    ├── OPEN TEACH          Meta Quest 3 ~$500，低成本通用 VR 框架
    │
    └── TWIST2              PICO 4 + 全身 tracker，无 mocap
                            Unitree G1 whole-body fleet 规模化
```

### 4.3 A vs C 对比

| | ALOHA Teleop | VR Teleop |
|--|-------------|-----------|
| 映射 | Leader 关节 **直接 copy** | VR pose → **IK** |
| 机器人 | 双臂桌面 | 人形 **whole-body** |
| 相机 | 固定俯视 + 腕部 | **头戴主动立体** |
| 成本 | ~$20k | OTV 高；OPEN TEACH ~$500 起 |
| 精度 | 极高（关节级） | IK/retarget 有误差 |

**笔记**：[VR 与人形 Teleop 数采](./VR与人形Teleop数采.md) · [Open-TeleVision 精读](./Open-TeleVision-Paper-Walkthrough.md)

## 五、范式 D · Ego / 人类视频

### 5.1 核心采集方法

```text
普通第一人称视频（眼镜/手机/互联网）
    ↓
只有 RGB（+ 可选字幕），无 robot action
    ↓
必须「翻译」才能训练：
  · 3D hand tracking → retarget
  · inpainting 人手→机械手
  · latent action（不直接 motor）
  · 仿真重建（Video2Policy）
```

| 项 | 内容 |
|----|------|
| **采集时要机器人** | ❌ |
| **原始数据** | MP4 |
| **Action** | **无**；需估计 / 隐式表示 |
| **Fidelity** | ⭐⭐ |
| **Scale** | **极大**（Ego4D 3K+ hr，EgoScale 20K+ hr） |

### 5.2 四条对齐路径

| 路径 | 代表 | 输出 |
|------|------|------|
| **物理 Retarget** | EgoMimic, EgoBridge（OT 对齐） | 可行 EEF / latent 对齐 |
| **视觉 Inpainting** | H2R, DexUMI | 机械手视角图像 |
| **Latent Action** | Phantom, EgoVLA | 预训练 prior，非直接 motor |
| **结构重建** | Video2Policy | 仿真任务代码 → rollout |

### 5.3 演进树

```text
数据规模线:
  Ego4D → EgoScale (20K+ hr, scaling law)

VLA 接入线:
  EgoMimic → EgoVLA → Phantom / EgoZero（零 robot 探索）

域对齐线:
  EgoBridge（OT 对齐 human↔robot latent, +44%）

人形扩展线:
  EgoHumanoid, EgoMI, EMMA, HAT

理论线:
  Human→Robot Transfer in VLA（何时涌现迁移）
```

**笔记**：[Ego 视频数采与对齐](./Ego视频数采与对齐.md) · 论文索引 §5

## 六、四维对比总表

| 维度 | **A Teleop** | **B UMI** | **C VR** | **D Ego** |
|------|-------------|-----------|----------|-----------|
| **采集时要机器人** | ✅ | ❌ | ✅ | ❌ |
| **Action 来源** | 关节编码器（真） | SLAM/追踪（估） | IK retarget | 手追踪/latent/重建 |
| **相机** | 固定+腕部 | 仅腕部 | 头戴主动立体 | 人眼视角 |
| **Action 表示** | 14D joint | 7D relative EEF | whole-body joint | 需翻译 |
| **典型成本** | $20k+ | $1k 级 | $500–高 | 最低 |
| **Fidelity** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |
| **Scale 天花板** | 小 | 中→大 | 中 | **极大** |
| **部署对齐** | 同机器人同相机 | 同构夹爪+腕部相机 | 同人形+头显相机 | 需 finetune 校准 |
| **Data Pyramid Tier** | 2–3 | 2–3 | 2–3 | **1** |

## 七、原始数据 → 训练数据管线

```text
┌─────────────┬──────────────┬─────────────┬──────────────┐
│ A Teleop    │ B UMI        │ C VR        │ D Ego        │
├─────────────┼──────────────┼─────────────┼──────────────┤
│ 原始:       │ 原始:        │ 原始:       │ 原始:        │
│ MCAP/ROS    │ GoPro MP4    │ VR pose 流  │ 人类 MP4     │
│ 或 HDF5直录 │ + IMU        │ + 头显视频  │              │
├─────────────┼──────────────┼─────────────┼──────────────┤
│ 处理:       │ 处理:        │ 处理:       │ 处理:        │
│ 同步<10ms   │ SLAM/T265    │ IK/retarget │ hand track   │
│ QA 过滤     │ relative EEF │ latency对齐 │ inpaint/OT   │
│             │ latency匹配  │             │ latent       │
├─────────────┼──────────────┼─────────────┼──────────────┤
│ 导出:       │ 导出:        │ 导出:       │ 导出:        │
│ HDF5/LeRobot│ zarr/自定义  │ LeRobot     │ 弱/强监督    │
│ RLDS(可选)  │ LeRobot      │             │ 多 stage     │
└─────────────┴──────────────┴─────────────┴──────────────┘
                           ↓
                    VLA / ACT / DP 训练
```

## 八、选型决策树

```text
要毫米级双臂精细操作、可复现 ACT/RDT？
  → A: ALOHA / ALOHA 2

有 Franka/UR5，不想买 leader 臂？
  → A: GELLO

要移动+双臂（做饭、开柜）？
  → A: Mobile ALOHA 或 TidyBot++

不想带真机去厨房/野外？
  → B: UMI → FastUMI →（规模化）RDT2

要灵巧手多指操作？
  → B: DexUMI（或 A: DexCap mocap）

要人形 whole-body、沉浸式感知？
  → C: Open-TeleVision / OPEN TEACH / TWIST2

只有人类视频、要冲预训练 scale？
  → D: EgoScale + EgoVLA，但 Tier 3 必须补 A 或 B
```

## 九、混训注意事项

```text
❌ 不要 raw 混：ALOHA joint + UMI EEF + Ego latent 直接同一 batch
✅ 要：Canonical IR → 统一 action 空间 → embodiment_id → 采样权重
✅ Data Pyramid：Ego(Tier1) → OXE/UMI(Tier2) → Teleop(Tier3)
```

## 十、相关训练概念速查

| 概念 | 与数采关系 |
|------|-----------|
| **Co-train** | 多数据源同一阶段混合训练（如 Mobile ALOHA 静态+移动） |
| **DAgger** | 交互式 IL：策略跑偏状态上补专家标注；真机成本高，ALOHA 多用 ACT chunking 代替 |
| **Unified Action** | 多源混训前映射到统一空间（RDT 128-d） |

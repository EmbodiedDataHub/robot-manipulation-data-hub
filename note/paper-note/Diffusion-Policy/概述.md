## 一句话

Diffusion Policy 把 **DDPM 扩散模型** 用在 **动作轨迹** 上：给定最近几步观测，从噪声里 **逐步去噪** 出未来 `horizon` 步动作序列，再按 **receding horizon** 只执行前 `n_action_steps` 步——能建模人类 demo 的 **多模态**，在 Push-T、Robomimic、真机 Push-T 等任务上显著优于 BC-RNN / IBC / BET。

## 核心概念速查

| 概念 | 含义 |
|------|------|
| **horizon (T)** | 一次扩散预测的动作轨迹长度（如 16 步） |
| **n_obs_steps (To)** | 条件化用的历史观测帧数（如 2 步） |
| **n_action_steps** | 每次 rollout 实际执行的动作步数（如 8 步） |
| **receding horizon** | 每 `n_action_steps` 重新采样一段新轨迹，只执行开头几步 |
| **global cond** | 观测编码成向量，作为 UNet 全局条件（Push-T image 默认） |
| **inpainting** | 把观测特征拼进轨迹 tensor，用 mask 固定已知部分 |
| **epsilon prediction** | 网络预测加入的噪声 ε，与 Stable Diffusion 同类 |
| **EMA** | 训练时对权重做指数滑动平均，评估时用 EMA 模型 |

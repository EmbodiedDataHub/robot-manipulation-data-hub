# 机器人操作数据 · 快速入门（通俗版）

> 写给**零基础 / 非机器人背景**的同学：少公式、多类比、中文优先。  
> 目标：7 天能看懂 ALOHA / UMI / RDT 在讲什么，不必先啃完一门课。

## 先读这段：你在学什么？

想象教小孩叠杯子：

1. **你亲手示范**（人类演示 / Teleop）
2. **录下视频 + 手怎么动**（数据集）
3. **AI 学会：看到杯子 → 输出动作**（策略 / Policy）
4. **机器人自己叠**（部署）

整条技术线就是在回答：**怎么录、怎么存、怎么训、怎么让不同机器人都能用**。

```
ALOHA      = 在实验室用真实机械臂录 demo（贵但准）
UMI        = 拿手持夹爪去厨房录 demo（不用带机器人）
Open-TeleVision = 戴 VR，像「灵魂附体」到人形机器人上录 demo
RDT        = 把海量 demo 喂给大模型，学会通用操作
```

## Day 0 · 机器人基本概念（0.5–1 天，建议先过一遍）

> 关节角、末端位姿这些词**不是数学炫技**，就是在说：**机器人的动作到底怎么记录**。  
> 不用学 DH 公式推导，知道「是什么、论文里为啥选它」就够。

### 先认识机械臂（30 秒）

```
底座 → 关节1 → 关节2 → … → 手腕 → 夹爪（末端执行器）
         ↑                    ↑
      每个关节转多少度      夹爪在空间中的位置+朝向
      = 关节角               = 末端位姿 (EEF pose)
```

| 词 | 人话 | 论文里常见写法 |
|----|------|----------------|
| **DoF（自由度）** | 能独立控制的方向有几个 | 6 轴臂 = 6 DoF；夹爪 +1 |
| **Joint / 关节角** | 每个关节转了多少度 | ALOHA 直接录 14 个数（双臂） |
| **EEF / 末端执行器** | 夹爪/手 — 真正干活的那端 | end-effector |
| **EEF pose / 末端位姿** | 夹爪在空间 xyz + 朝向 | 6 个数（位置 3 + 旋转 3）+ 夹爪开合 |
| **Proprioception / 本体感** | 机器人「感觉」自己关节在哪 | 除相机外的 joint state |
| **Teleop / 遥操作** | 人远程操控并录 demo | leader-follower、VR、手持设备 |

### 四种「动作写法」（读 UMI/RDT 必懂）

| 写法 | 记录的是啥 | 类比 | 谁爱用 |
|------|-----------|------|--------|
| **关节角** | 每个关节转几度 | 记「肘弯 30°、腕转 15°」 | ALOHA（leader 臂直接抄关节） |
| **末端位姿** | 夹爪在空间的 xyz + 朝向 + 开合 | 记「手在桌子左上角、朝下」 | 大多数 manipulation 论文 |
| **相对位移** | 比**上一帧**挪了多少 | 记「向左 2cm、往下 1cm」 | **UMI**（定位会漂，相对更稳） |
| **统一动作空间** | 不同机器人翻译成同一套「物理语言」 | 普通话 vs 各地方言 → 统一书面语 | **RDT**（混多种机器人数据） |

**正/逆运动学（知道名字即可，不必算）：**
- **正运动学 FK**：关节角 → 末端在哪（给定角度，算夹爪位置）
- **逆运动学 IK**：末端要去哪 → 各关节该转多少（Open-TeleVision 用人手 pose → IK → 机器人关节）

## Day 1 · 模仿学习 = 「照着做」

### 一句话

| 概念 | 类比 |
|------|------|
| 专家演示 | 师傅教你拧螺丝，你在旁边看 |
| 数据集 | 把「看到了啥 + 手怎么动」存成文件 |
| 策略 π | 学成之后你自己上手，不用师傅在场 |
| 部署 | 机器人 autonomous 执行 |

## Day 2 · 为啥「学一步错一步」

### 一句话

机器人每动一下都有小误差，误差叠在一起，就像**走迷宫越走越偏**，最后完全不像训练时的样子。这叫 **compounding error（误差累积）**。

### 生活类比

```
走直线贴墙：每步偏 1cm → 走 100 步就偏 1 米，撞墙

开瓶盖：第 1 步偏了 2mm → 后面每一步都在「错误状态」上决策 → 盖子弹飞
```

ALOHA 的解法：**一次规划未来好几步（Action Chunking）**，而不是「只看眼前这一步」。

### 推荐资料

| 优先级 | 资料 | 说明 |
|--------|------|------|
| ⭐1 | 读本文件 Day 2 类比 + ALOHA 中文 PDF **Related Work** 里 compounding 那段 | 10 分钟 |
| ⭐2 | [Mobile ALOHA 中文官网](https://mobile-aloha.github.io/cn.html) | 中文，看「遥操作 → 自主执行」对比图 |
| 进阶 | [CS285 Lecture 2–3](http://rail.eecs.berkeley.edu/deeprlcourse/) | 英文好再看；**非必须** |

### 今日小测

- 为啥精细操作（插电池）比「抓大块」更难学？（误差容忍度低）
- Action Chunking 直觉上在干什么？（一次想好一小段路，不是一步一蹭）

## Day 3 · 动作有四种「写法」

### 一句话

同一套「手往左移 5cm」，可以记录成：**关节角度变了多少**、**手腕在空间的位置**、**相对上一帧挪了多少**……  
写法不同，后面能不能迁到别的机器人上，差别很大。

### 对照表（背这张就够读 UMI/RDT）

| 写法 | 人话 | 谁爱用 | 优点 | 缺点 |
|------|------|--------|------|------|
| **关节角** | 每个关节转几度 | ALOHA | 直接控制机器人 | 换臂长就不通用 |
| **末端位姿** | 手在空间 xyz + 朝向 | 大多数论文 | 直观 | 要准的定位 |
| **相对位移** | 比上一帧挪多少 | **UMI** | 定位误差影响小 | 要连贯轨迹 |
| **统一动作空间** | 不同机器人翻译成同一套「物理语言」 | **RDT** | 能混合多机器人数据 | 设计复杂 |

### 推荐资料

| 优先级 | 资料 |
|--------|------|
| ⭐1 | 本地 UMI 中文/英文 PDF — **只看 Fig.2（手持设备长啥样）** |
| ⭐2 | `机器人操作数据学习路线报告.md` §二 对比表 |
| 选看 | RDT-1B PDF — Unified Action Space 那一节（知道「为了混数据」即可） |

## Day 4 · Diffusion Policy = 「从噪声里抠出合理动作」

### 一句话

人演示同一件事，**可以左绕也可以右绕**（多模种路线）。  
普通神经网络会取「平均路线」→ 撞墙。  
**Diffusion Policy** 像 Stable Diffusion 画图画，从噪声里**慢慢修**出一条合理动作轨迹。

### 生活类比

```
任务：把 T 字块推到目标位置

普通 BC：学出「走中间」→ 中间有障碍就傻住
Diffusion：知道「可以左绕或右绕」→ 采样一条能走的
```

### 推荐资料（中文优先，不用先读原文）

|--------|------|------|
| ⭐1 | [简单理解 Diffusion Policy（中文博客）](https://deathgarurumon.github.io/docs/diffusion_policy/) | 短、有图，**Day 4 主材料** |
| ⭐2 | [Diffusion Policy 论文精读（Devon Blog）](https://devon018.github.io/Diffusion_Policy/) | 稍长，想深入再看 |

## Day 5 · ACT = 「一次想好接下来几步」

### 一句话

**ACT（Action Chunking with Transformers）** = ALOHA 自带的算法：  
看当前画面 → **一次输出未来 k 步动作** → 减少「走一步偏一步」。

### 三个词（不用记公式）

| 词 | 人话 |
|----|------|
| **Chunking** | 一次规划一小段，不是只规划下一步 |
| **CVAE** | 演示里人有停顿、路径不同，用隐变量表示「这种随机性」 |
| **Temporal Ensembling** | 相邻两次预测有重叠，取平均 → 动作更顺滑 |

### 推荐资料

| 优先级 | 资料 |
|--------|------|
| ⭐1 | [Bilibili：ACT / ALOHA 精析](https://www.bilibili.com/video/BV1xGF3eeEjB) — **~42 min 中文精讲 Chunk/CVAE/TE** |
| ⭐2 | [ALOHA · ACT 笔记集概述](./paper-note/ALOHA/概述.md) — 论文 + 原理 + 代码导读总入口 |
| ⭐3 | [ALOHA 官网视频](https://tonyzhaozh.github.io/aloha) — 看学前后对比 |
| ⭐3 | 本地 ALOHA **Method** 一节（中文 PDF） |
| ⭐3 | [LeRobot ACT 训练示例](https://hugging-face.cn/docs/lerobot/il_robots) | 知道 `--policy=act` 即可 |

- ACT 和 Diffusion Policy 都在解决啥问题？（动作要连贯、多样、少累积误差）
- Open-TeleVision 也用 ACT —— 说明 ACT 是「通用配方」，不绑 ALOHA 硬件

## 先读这段：你在哪个大故事里？

想象你在教一个**刚出生的 AI 小孩**做家务：

```
第 1 代：只会「照猫画虎」—— 看师傅怎么做，它模仿（模仿学习）
第 2 代：会「举一反三」—— 看过很多物体，新杯子也会抓（泛化）
第 3 代：能「听懂人话」—— 「把红色杯子放到左边」它知道干啥（语言条件）
第 4 代：像 ChatGPT 一样「通才」—— 多种机器人、多种任务、一种模型（Foundation Model / VLA）
```

**VLA（Vision-Language-Action，视觉-语言-动作模型）** 就是第 4 代的代表：

> **眼睛看画面 + 耳朵听指令 → 输出机器人怎么动**

它和 ChatGPT 的关系：

| | ChatGPT | VLA |
|---|---------|-----|
| 输入 | 文字 | **图片/视频 + 文字指令** |
| 输出 | 文字 | **动作**（关节转几度、手往哪移） |
| 训练数据 | 互联网文本 | **机器人演示 + 人类视频 + 仿真** |
| 类比 | 会写作文的大脑 | 会干活的「具身大脑」 |

## 一、整体地图：五层楼模型

把机器人 AI 想成一栋楼，从下到上：

```
┌─────────────────────────────────────────────────────────────┐
│  5F · VLA 大模型（通才）                                      │
│  RT-2 · π0 · OpenVLA · RDT · Qwen-Robot · Gr00t …           │
│  「一种模型，多种机器人，听懂人话，做各种活」                    │
└─────────────────────────────────────────────────────────────┘
                              ↑ 需要海量、多样、高质量数据
┌─────────────────────────────────────────────────────────────┐
│  4F · 数据基础设施                                            │
│  采集(Teleop/UMI/Ego) · 格式(RLDS/LeRobot) · Data Engine     │
│  「原材料从哪来、怎么洗、怎么喂给模型」                          │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│  3F · 策略算法（Policy）                                      │
│  BC · ACT · Diffusion Policy · 3D Diffusion · Flow Matching  │
│  「看到画面后，用什么数学方式决定动作」                          │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│  2F · 机器人硬件 & 控制                                       │
│  机械臂 · 夹爪 · 相机 · 力传感器 · 控制器(50Hz) · IK/FK       │
│  「身体长什么样、怎么精确动起来」                                │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│  1F · 任务 & 评估                                             │
│  抓杯子 · 开抽屉 · LIBERO · CALVIN · 成功率 · 泛化测试         │
│  「什么叫做好了、怎么打分」                                      │
└─────────────────────────────────────────────────────────────┘
```

**你现在学的「机器人数据」，主要在 4F，但 4F 撑起了 5F。**  
没有好数据，VLA 就像 ChatGPT 只有几本书——能聊，但干不了活。

## 二、机器人必备基础知识（30 分钟版）

> 详细版见 [快速入门 Day 0](./快速入门-最短学习路径.md#day-0--机器人基本概念051-天建议先过一遍)

### 2.1 机械臂长什么样？

```
底座 → 关节1 → 关节2 → … → 手腕 → 夹爪（末端执行器 EEF）
         ↑                    ↑
      每个关节转多少度      夹爪在空间的位置 + 朝向
      = 关节角 (Joint)      = 末端位姿 (EEF Pose)
```

| 词 | 人话 | 为什么重要 |
|----|------|-----------|
| **DoF（自由度）** | 能独立控制的方向有几个 | 6 轴臂 = 6 DoF；夹爪开合 +1 |
| **EEF / 末端执行器** | 真正干活的「手」 | 论文里 90% 的动作都围绕它描述 |
| **Proprioception / 本体感** | 机器人「感觉」自己关节在哪 | 除相机外的 joint state |
| **Teleop / 遥操作** | 人远程操控并录 demo | VLA 训练数据的头号来源 |
| **Embodiment / 具身形态** | 机器人长什么样 | 换臂长、换夹爪 = 换 embodiment |

### 2.2 动作的四种「写法」（读任何 VLA 论文必懂）

同一套「手往左移 5cm」，记录方式不同，能不能跨机器人复用，差别巨大：

| 写法 | 记录的是啥 | 类比 | 谁爱用 |
|------|-----------|------|--------|
| **关节角** | 每个关节转几度 | 记「肘弯 30°、腕转 15°」 | ALOHA |
| **末端位姿** | 夹爪 xyz + 朝向 + 开合 | 记「手在桌子左上角、朝下」 | RT-1、DROID、大多数论文 |
| **相对位移 (Delta)** | 比**上一帧**挪了多少 | 记「向左 2cm、往下 1cm」 | UMI、BridgeData |
| **统一动作空间** | 不同机器人翻译成同一套「物理语言」 | 各地方言 → 普通话 | RDT、Open X-Embodiment |

**正/逆运动学（知道名字即可）：**
- **FK 正运动学**：关节角 → 手在哪
- **IK 逆运动学**：手要去哪 → 各关节该转多少

### 2.3 闭环 vs 开环（为什么机器人会「越走越偏」）

```
开环：看一眼 → 规划一整段动作 → 闭眼执行 → 误差累积 → 撞墙
闭环：每 50ms 看一次 → 微调 → 再看 → 微调 ……（像开车不断修正方向）
```

| 概念 | 人话 |
|------|------|
| **Compounding error** | 每步偏一点，100 步后偏很多（走迷宫越走越歪） |
| **Action Chunking** | 一次规划未来好几步，而不是一步一蹭（ACT 的核心思路） |
| **Control frequency** | 控制频率，工业 VLA 常见 **50 Hz**（每秒决策 50 次） |
| **Hz** | 1 Hz = 每秒 1 次；50 Hz = 每秒 50 次 |

### 2.4 一条 demo 数据里到底有什么？

```
一帧 (timestep) = {
  观测 (Observation):  相机画面 + 关节状态 + 有时还有语言指令
  动作 (Action):         下一步该怎么动（关节角 / EEF / delta …）
}

一条轨迹 (Episode) = 从任务开始到结束的很多帧
一个数据集 = 成千上万个 Episode
```

**训练 VLA 的本质**：给 `(画面, 语言)` → 预测 `动作`，跟 ImageNet 训分类器一样，只是标签换成了「怎么动」。

### 2.5 仿真 vs 真机

| | 仿真 (Sim) | 真机 (Real) |
|---|-----------|------------|
| 优点 | 便宜、快、可并行、不怕摔 | 真实物理、真实传感器噪声 |
| 缺点 | 物理不够真（Sim-to-Real gap） | 贵、慢、一台机器人一次只能干一件事 |
| 代表 | MuJoCo, Isaac Sim, RoboCasa | ALOHA, Franka, WidowX |
| 趋势 | 仿真生成海量数据 + 少量真机微调 | **Sim + Real 混合** 是主流 |

## 三、VLA 发展历程：从「照猫画虎」到「具身 ChatGPT」

### 3.1 时间线总览

```
2016–2019  │ 深度学习进机器人：端到端 BC、DAgger
           │ 「摄像头 → 神经网络 → 关节角」，概念验证期
           │
2020–2022  │ 模仿学习成熟期
           │ BC 误差累积问题被正视；Sim-to-Real；多任务数据集
           │
2023       │ ★ 转折年：Diffusion Policy + 大模型联姻
           │ Diffusion Policy：解决「多模态动作」问题
           │ RT-1 / RT-2 (Google)：把 VLM 搬进机器人
           │ Open X-Embodiment：22 种机器人数据大合集
           │
2024       │ VLA 爆发年
           │ π0 (Physical Intelligence)：Flow Matching + 大规模数据
           │ OpenVLA、Octo、RDT-1B：开源 VLA 军备竞赛
           │ UMI / Mobile ALOHA：数据采集成本骤降
           │
2025–2026  │ 数据基础设施时代
           │ 行业共识：瓶颈从「模型不够大」→「数据不够好」
           │ Data Engine、Ego 视频、世界模型、跨 embodiment 预训练
           │ Qwen-Robot、Gr00t N1、RDT2 … 工业级 VLA 落地
```

### 3.2 每个阶段用人话解释

#### 阶段 1：端到端模仿学习（2016–2022）

**核心思想**：别手写规则，让神经网络自己学「看到什么 → 怎么动」。

**代表**：Behavior Cloning (BC)、DAgger、QT-Opt（抓取）

**痛点**：
- 误差累积：学一步错一步，精细操作（插 USB）几乎不可能
- 数据少：每个任务都要人手把手教
- 泛化差：换个杯子颜色就不认识了

**生活类比**：像只看过 10 次师傅拧螺丝的学徒——螺丝位置稍微变一点就懵。

---

#### 阶段 2：Diffusion Policy（2023，Columbia / Toyota Research）

**解决了什么**：同一个任务可以**左绕也可以右绕**（多模态），普通 BC 会学出「走中间」然后撞墙。

**核心直觉**：像 Stable Diffusion 从噪声里「修」出一张图，Diffusion Policy 从噪声里「修」出一条合理动作轨迹。

**影响**：UMI、ALOHA 2、很多后续 VLA 的动作头都用 Diffusion 或它的变体。

**生活类比**：普通 BC = 只记住一条固定路线；Diffusion = 知道「能左绕能右绕」，每次采样一条能走的。

---

#### 阶段 3：RT-1 / RT-2 — VLA 的「开山之作」（Google DeepMind, 2022–2023）

**RT-1 (Robotics Transformer 1)**：
- 700+ 任务、130K demo、车队级遥操作
- 纯机器人数据训练的 Transformer
- 证明了：**一个模型可以做多任务**

**RT-2 (Robotics Transformer 2)** — **VLA 这个名字真正火起来**：
- 把 **PaLI（视觉-语言模型）** 和 RT-1 合在一起
- 关键创新：**把动作离散化成 token**，跟文本 token 一起训练
- 效果：机器人能利用互联网上的视觉-语言知识（「识别 extinct animal」→ 拿起对应玩具）

**公式（不用记，感受逻辑）**：

```
RT-2 输入:  图片 + 「pick up the extinct animal」
RT-2 输出:  一串 token → 解码成机器人动作

=VLM 的「看懂 + 听懂」+ 机器人数据的「会做」
```

**生活类比**：RT-1 是只会干活的技工；RT-2 是读过百科全书的技工——你说「灭绝动物」，它知道长什么样。

---

#### 阶段 4：Open X-Embodiment — 数据大统一（2023, Google + 33 机构）

**解决了什么**：每个 lab 用自己的机器人、自己的数据格式，模型学不到「通用操作」。

**做了什么**：把 **22 种机器人、527 技能、160 万条轨迹** 统一格式（RLDS），开源给社区。

**意义**：像 ImageNet 对 CV 的意义——没有它，就没有后来的大规模预训练。

**生活类比**：以前每个省说方言；OXE 制定了「普通话」语法，大家的数据能混一起训了。

---

#### 阶段 5：ACT + ALOHA — 低成本真机数据采集（2023–2024, Stanford）

**ALOHA**：$20K 造双臂遥操作台， fidelity 极高  
**ACT (Action Chunking with Transformers)**：一次输出未来 50–100 步动作，解决误差累积

**Mobile ALOHA**：加上移动底盘，能做家务（洗锅、浇花）

**意义**：证明了「**数据质量 > 数据 fancy 算法**」——ALOHA 的 demo 干净、同步好，ACT 结构并不复杂，但效果惊艳。

---

#### 阶段 6：UMI — 不用机器人也能采数据（2024, Columbia / Stanford）

**核心创意**：拿一个**手持夹爪 + GoPro** 去真实厨房录 demo，SLAM 推断出「假如是机器人在做，该怎么动」。

**意义**：数据采集成本从「买一台 Franka（$30K+）」降到「一个夹爪 + 相机（$1K 级）」。

**和 VLA 的关系**：UMI 数据可以喂给任何 Policy / VLA，是 **Data Engine** 路线的典型代表。

---

#### 阶段 7：Foundation Model 军备竞赛（2024–2026）

| 模型 | 机构 | 一句话 |
|------|------|--------|
| **π0** | Physical Intelligence | Flow Matching + 海量跨机器人数据，工业级 dexterous 操作 |
| **OpenVLA** | Stanford 等 | 开源 7B VLA，基于 Llama + SigLIP |
| **Octo** | Berkeley 等 | 开源通用策略，支持多种机器人 finetune |
| **RDT-1B / RDT2** | 清华等 | 统一 Action Space + 扩散 Transformer，中文社区友好 |
| **Gr00t N1** | NVIDIA | 人形机器人 VLA，whole-body 控制 |
| **Qwen-Robot** | 阿里 | 大规模中文机器人数据 + VLA |
| **EgoVLA / H2R** | 多机构 | 从人类第一人称视频学操作 |

**2025–2026 行业共识**（见 [数据调研报告](./机器人数据工作综合调研报告.md)）：

> VLA 下一波突破，更依赖 **Data Engine + Evaluation** 的协同设计，而非单纯堆模型参数。

## 四、VLA 内部长什么样？（解剖图，不用懂公式）

一篇 VLA 论文通常拆成四块：

```
┌─────────────────────────────────────────────────────────┐
│  1. Vision Encoder（眼睛）                               │
│     SigLIP / ViT / DINOv2 — 把图片变成向量                │
├─────────────────────────────────────────────────────────┤
│  2. Language Encoder（耳朵）                             │
│     LLaMA / T5 / Qwen — 把「把杯子放左边」变成向量         │
├─────────────────────────────────────────────────────────┤
│  3. Fusion / Backbone（大脑融合层）                       │
│     Transformer — 把视觉 + 语言 + 有时还有本体感 融合      │
├─────────────────────────────────────────────────────────┤
│  4. Action Head（手）                                    │
│     离散 token (RT-2) / Diffusion (π0, RDT) / MLP / Flow │
│     输出：未来 H 步动作（Action Chunk）                    │
└─────────────────────────────────────────────────────────┘
```

**两种动作输出方式（知道区别即可）：**

| 方式 | 代表 | 直觉 |
|------|------|------|
| **离散 token** | RT-2 | 把动作切成 256 个「字母」，跟文本一起生成 |
| **连续 diffusion / flow** | π0, RDT, Diffusion Policy | 从噪声里「修」出平滑轨迹，适合精细操作 |

## 五、数据采集：VLA 的「燃料」从哪来？

五条路，从贵到便宜、从准到糙：

```
质量/保真度
  ★★★★★ │ A. 真机 Teleop（ALOHA, DROID, RT-1）
         │    人操作真机器人，动作是「真的做了」的
  ★★★★   │ B. 仿真生成（MimicGen, RoboCasa, RoboGen）
         │    200 条人类 demo → 算法自动生成 5 万条
  ★★★    │ C. 手持野外（UMI, FastUMI, DexUMI）
         │    不用机器人，GoPro + 夹爪，SLAM 推断动作
  ★★     │ D. 人类 Ego 视频（Ego4D, EgoVLA, H2R）
         │    第一人称做饭视频，没有动作标签，要「翻译」
  ★      │ E. 互联网视频（Video2Policy, RoboWheel）
         │    YouTube 视频，最难对齐，但量最大
         │
         └──────────────────────────────────→ 规模/可扩展性
```

**2026 趋势**：不是单选，而是 **Data Pyramid（数据金字塔）**——

```
        ┌─────────┐
        │ 少量高质量 │  真机 Teleop，精调
        │  真机数据  │
        ├─────────┤
        │ 中量混合  │  UMI + 仿真 + 跨机器人 OXE
        ├─────────┤
        │ 海量弱监督 │  Ego 视频 + 互联网视频，预训练
        └─────────┘
```

## 六、怎么评估一个 VLA 好不好？

| 层级 | 评什么 | 代表 Benchmark |
|------|--------|----------------|
| 简单桌面 | 抓、放、推、开抽屉 | LIBERO, Meta-World |
| 语言条件 | 听懂指令做操作 | CALVIN, VLABench |
| 长时域 | 多步任务链 | BEHAVIOR-1K |
| 泛化/鲁棒 | 换物体、换光照、加干扰 | LIBERO-Plus, SimplerEnv |
| 跨机器人 | 换 embodiment 还能用 | Open X-Embodiment eval |

**常见坑**：训练集上 95% 成功率，换个背景/物体就掉到 30%——所以 2026 年强调 **诊断性 benchmark**，不只报一个 success rate。

## 七、关键概念速查表

| 词 | 一句话 |
|----|--------|
| **VLA** | 看图 + 听指令 → 输出机器人动作的大模型 |
| **VLM** | 视觉-语言模型，VLA 的「上半截」 |
| **Policy / 策略** | 看到什么 → 怎么动的函数 π(a\|s) |
| **BC / 行为克隆** | 监督学习式模仿：复制专家动作 |
| **IL / 模仿学习** | 从演示中学习策略的总称 |
| **Foundation Model** | 大规模预训练、可 finetune 的通用模型 |
| **Finetune / 微调** | 在大模型基础上，用本实验室数据再训 |
| **Pretrain / 预训练** | 用海量混合数据先训一个「通才」 |
| **Sim-to-Real** | 仿真训、真机部署，中间有 domain gap |
| **Data Engine** | 自动化流水线：采集 → 清洗 → 标注 → 训练 |
| **RLDS** | Google 的机器人数据集标准格式 |
| **LeRobot** | Hugging Face 的机器人 ML 开源栈 |
| **Action Chunk** | 一次预测未来多步动作（ACT/π0 常用） |
| **Co-training** | 机器人数据 + 互联网 VLM 数据一起训 |
| **World Model** | 学「世界怎么运转」，可仿真推演（3D-VLA, UniSim） |
| **Dexterous** | 灵巧手操作（比平行夹爪难一个数量级） |
| **Bimanual** | 双臂协作（ALOHA 专长） |
| **Whole-body** | 人形机器人全身控制（不只是手臂） |

## 九、5 分钟自测（能答就入门了）

1. **VLA 和 ChatGPT 最大的区别是什么？**  
   → 输出从「文字」变成「动作」；输入多了「摄像头画面」

2. **RT-2 为什么比 RT-1 更「聪明」？**  
   → 融合了 VLM 的互联网视觉-语言知识，能理解更抽象的概念

3. **为什么 UMI 不用真机器人也能采数据？**  
   → 用 SLAM 追踪手持设备运动，推断出「假如是机器人在做」的轨迹

4. **2026 年 VLA 的核心瓶颈是什么？**  
   → 数据质量、多样性、评估协议（不只是模型大小）

5. **Action Chunking 解决什么问题？**  
   → 误差累积——一次规划多步，而不是每步独立决策

6. **关节角和末端位姿，哪个更「绑具体机器人」？**  
   → 关节角（换臂长就不通用）

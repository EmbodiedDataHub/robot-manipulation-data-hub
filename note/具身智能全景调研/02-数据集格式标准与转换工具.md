# 机器人数据集格式标准与转换工具

> **核心问题**：机器人数据有哪些「事实标准」？各格式 schema 是什么？用什么库互转？有哪些教程？

---

## 一、一句话

```text
预训练/OXE  → RLDS（Episode/Step + TFRecord）
HF 微调     → LeRobot（Parquet + MP4）
传统 IL     → HDF5（每 episode 一个文件）
真机录制    → MCAP/ROS bag → Canonical IR → 导出 RLDS 或 LeRobot
```

**训练接口最终都收敛为**：`(观测 o_t, 语言 ℓ, 动作 a_t)`，差别在**磁盘布局与生态**。

---

## 二、格式对比总表

| 格式 | 存储 | 生态 | 典型用途 | 优点 | 缺点 |
|------|------|------|----------|------|------|
| **RLDS** | TFRecord | TF/JAX/Octo/OpenVLA | OXE 预训练 | 异构 dim 可共存、大规模混合 | TF 倾向、本地 PyTorch 需适配 |
| **LeRobot** | Parquet + MP4 | HF/PyTorch/lerobot-train | 真机微调、中小规模 | 易上手、HF Hub 一键拉 | TB 级全量 OXE 转换仍在推进 |
| **HDF5** | `.hdf5` | ACT/robomimic/LIBERO | 仿真 benchmark、双臂 | 成熟、数组直读 | 跨项目 schema 不统一 |
| **MCAP/ROS** | bag/mcap | ROS2 真机 | 原始录制 | 工业标准 | 不直接喂 VLA，需 QA+导出 |

## 三、RLDS / TFDS（Open X-Embodiment 标准）

### 3.1 逻辑 schema

```text
Dataset
 └── Episode
      ├── episode_metadata  (robot_type, file_path, ...)
      └── steps[]  (按时间排序)
           ├── observation
           │    ├── image_*     (JPEG bytes 或 uint8[H,W,3])
           │    ├── state       (float32[D_state], 可选)
           │    └── ...
           ├── action           (float32[D_action] 或 action_dict)
           ├── language_instruction  (string, 可选)
           ├── is_first / is_last / is_terminal
           └── discount / reward (RL 扩展, 多为 0)
```

### 3.2 关键文件

| 文件 | 作用 |
|------|------|
| `dataset_info.json` | 版本、描述、citation |
| `features.json` | 每字段 dtype/shape（TFDS FeatureConnector） |
| `*.tfrecord-*` | 实际 episode 二进制 |

### 3.3 官方资源

| 资源 | 链接 |
|------|------|
| RLDS 规范 | [github.com/google-research/rlds](https://github.com/google-research/rlds) |
| OXE 工具链 | [google-deepmind/open_x_embodiment](https://github.com/google-deepmind/open_x_embodiment) |
| TFDS 文档 | [tensorflow.org/datasets](https://www.tensorflow.org/datasets) |

## 四、LeRobot（Hugging Face 生态）

### 4.1 目录结构（v2/v3 概念）

```text
dataset_root/
 ├── meta/
 │    ├── info.json          # fps, robot_type, features 定义
 │    ├── episodes.jsonl     # 每 episode 索引
 │    └── stats.json         # 归一化统计（可选）
 ├── data/
 │    └── chunk-*/file-*.parquet   # 逐步 tabular：state, action, timestamp
 └── videos/
      └── chunk-*/observation.images.*/*.mp4
```

### 4.2 特点

- **视频与表格分离**：Parquet 存数值，MP4 存多相机；通过 `timestamp` 对齐
- **HF Hub 集成**：`lerobot-train`、`LeRobotDataset` 直接 `from_pretrained`
- **v3 迁移**：官方 [porting datasets v3](https://huggingface.co/docs/lerobot/en/porting_datasets_v3) 文档

### 4.3 本地动手

- Notebook：[`01_lerobot_format.ipynb`](../../code/robot-dataset-demos/notebooks/01_lerobot_format.ipynb)
- 下载小样：`./scripts/download_samples.sh --lerobot-only`

## 五、HDF5（传统 IL）

### 5.1 典型 layout（LIBERO / ALOHA / robomimic）

```text
episode_0.hdf5
 ├── observations/
 │    ├── agentview_rgb      [T, H, W, 3]
 │    ├── wrist_rgb          [T, H, W, 3]
 │    └── qpos               [T, D]
 ├── actions                 [T, D_action]
 └── attrs: language, success, ...
```

### 5.2 使用场景

| 项目 | 为何仍用 HDF5 |
|------|--------------|
| LIBERO | Benchmark 历史包袱，社区工具链完整 |
| ACT / ALOHA | 原始采集即 HDF5，LeRobot 有转换版 |
| robomimic | Diffusion Policy 基线数据 |

## 六、Canonical IR（内部中间表示）

大型数据平台（EmbodiedDataHub、工业数采）常在导出前做 **Canonical IR**：

```text
原始 bag/HDF5/项目格式
    → 统一 episode/step 时间轴
    → 统一相机命名、action 物理语义（delta vs absolute）
    → QA 过滤、语言标注
    ├─→ RLDS exporter（贡献 OXE / 预训练）
    └─→ LeRobot exporter（HF 微调）
```

**为何需要 IR**：同一台 Franka 可能同时产出 ROS bag 和 HDF5；不经过 IR 直接互转容易丢字段或时间戳错位。

详见：[数采到VLA训练-数据管线整体方案 §六](../数采到VLA训练-数据管线整体方案.md)

### 7.2 any4lerobot 典型用法（概念）

```bash

```

### 7.3 Action 语义转换（最难部分）

格式转换 ≠ 语义对齐。常见需单独处理：

| 问题 | 例子 | 处理 |
|------|------|------|
| Delta vs Absolute | Bridge vs DROID | FK/差分或 per-dataset head |
| Joint vs EEF | ALOHA vs RT-1 | IK/FK 或 unified slot |
| 归一化 | OpenVLA quantile | 离线统计 per-dataset |
| 128 槽 unified | RDT | pad + mask |

详见：[预训练数据集清单与格式对齐 §三](../paper-note/Data-Pipeline/预训练数据集清单与格式对齐.md)

## 十、常见问题

**Q：RLDS 和 LeRobot 哪个是「标准」？**  
A：没有唯一标准。OXE 预训练事实标准是 RLDS；HF 微调生态快速收敛到 LeRobot。工业界常用 IR 双轨导出。

**Q：能否直接用 PyTorch 读 RLDS？**  
A：可以，通过 `tensorflow_datasets` 或 `dlimp`（Octo/OpenVLA 常用）；也可先转 LeRobot。

**Q：HDF5 会被淘汰吗？**  
A：Benchmark 与历史项目仍会长期存在；新采集更倾向 LeRobot。

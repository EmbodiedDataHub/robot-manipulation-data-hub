# 数采方法分类与详解

> **核心问题**：机器人演示数据有哪些采集范式？各范式 fidelity/scale/成本如何？如何进入 VLA 训练管线？

---

## 一、一句话

```text
四条主线：Teleop（真机高精度）→ UMI（无真机手持）→ VR 人形 → Ego 视频（规模化语义）
+ 仿真 fifth 线：LIBERO / RoboCasa / MimicGen bootstrap
最终都收敛为 (观测, 语言, 动作) → Canonical IR → RLDS / LeRobot
```

**深度版**：[数采四条主线全景图](../paper-note/Data-Pipeline/数采四条主线全景图.md)

---

## 二、分类总框架

```text
                    Fidelity ↑                    Scale ↑
                    ─────────────────────────────────────
  A. 实验室 Teleop     B. 手持 UMI          D. Ego 视频
  (ALOHA/DROID)        (无真机)              (普通人类视频)
        │                    │                    │
        │              C. VR / 人形 Teleop          │
        │              (whole-body)                 │
        └────────────────────┴──────────────────────┘
                              │
                    E. 仿真 / 合成（Bootstrap）
                              │
                    Canonical IR → 训练格式 → VLA
```

### Data Pyramid（与数采绑定的混合策略）

```text
Tier 3 · 精调：Teleop / UMI robot-aligned（决定部署成功率）
Tier 2 · 预训练：OXE / UMI 万小时 / 仿真扩增
Tier 1 · 先验：Ego / Web VLM（语义 breadth，不能单独 motor BC）
```

---

## 三、范式 A · 实验室 Teleop（真机主端从端）

### 3.1 原理

```text
人操作 Leader（主端）──关节/位姿映射──→ Follower（从端）真机
         │                                    │
         └────────── 同步录制 ─────────────────┘
              多相机 RGB + qpos/state + action + 时间戳
```

### 3.2 代表系统

| 系统 | 硬件 | Action 空间 | 典型规模 | 代表数据集 |
|------|------|-------------|----------|-----------|
| **ALOHA** | 双臂 ViperX Leader/Follower | Joint 14D | 50–500 ep/任务 | ALOHA sim/real |
| **DROID** | Franka + VR/GELLO 等 | Absolute EEF 7D | 76K ep | DROID |
| **GELLO** | 低成本主从 | Joint/EEF | 项目级 | 各 lab 自建 |
| **Mobile manipulator** | RT-1 车队 | Delta EEF + base | 130K ep | OXE fractal |

### 3.3 优缺点

| 优点 | 缺点 |
|------|------|
| **Action 与执行一致**（可部署 BC） | 成本高、规模受限 |
| 多相机、proprio 完整 | 场景多样性依赖 lab 布置 |
| VLA 微调 **Tier 3 主力** | 跨 lab schema 需 IR 对齐 |

### 3.4 工程要点

- **时间同步**：相机 vs joint state 对齐到统一时钟
- **Protocol**：固定相机位、指令模板、成功判定
- **导出**：HDF5（ACT）或 LeRobot（HF 微调）

**笔记**：[ALOHA 概述](../paper-note/ALOHA/概述.md) · [DROID 官网](https://droid-dataset.github.io/)

## 四、范式 B · UMI（Universal Manipulation Interface）

### 4.1 原理

```text
人手持夹爪 + GoPro（腕部相机）在真实环境操作
         ↓
SLAM/标定 → 轨迹重建 → Relative EEF action
         ↓
（可选）同构硬件部署到真机，零样本或少量适配
```

### 4.2 特点

| 维度 | 内容 |
|------|------|
| **无需真机采集** | 降成本、可进家庭/野外 |
| **Action** | Relative EEF 7D，跨平台语义稳定 |
| **规模** | RDT2：**10K+ 小时** 级 |
| **局限** | 无真实 joint 力矩；sim2real 靠硬件同构 |

### 4.3 与 Teleop 关系

- UMI **补 scale**；Teleop **补 deploy fidelity**
- RDT2 论证：UMI  alone 可支撑 **零样本 cross-embodiment**（需同构夹爪+相机）

**笔记**：[UMI](../paper-note/UMI-Universal-Manipulation-Interface.md) · [RDT2](../paper-note/RDT-Foundation-Models.md)

## 五、范式 C · VR / 人形 Whole-body Teleop

### 5.1 原理

```text
操作者 VR 头显 + 手柄/手套/外骨骼
         ↓
重定向到人形/双足/移动操作臂
         ↓
录制 whole-body joint / EEF + 多视角
```

### 5.2 代表方向

| 方向 | 场景 |
|------|------|
| **人形遥操作** | Optimus/Fourier 类 whole-body demo |
| **VR Franka** | DROID 采集协议之一 |
| **外骨骼 teleop** | 高精度臂操作 |

### 5.3 定位

- **Fidelity** 介于 Teleop 与 UMI 之间（沉浸但系统复杂）
- 2025–26 随人形机器人升温；数据格式仍多 **项目私有 → IR**

**笔记**：[VR与人形Teleop数采](../paper-note/Data-Pipeline/VR与人形Teleop数采.md)

## 六、范式 D · Ego 人类视频

### 6.1 原理

```text
第一人称 wearable 相机拍摄日常操作
         ↓
（无 robot action 标签）只有视频 + 弱标注
         ↓
用于：表征预训练 / 手物交互检测 / human→robot 对齐
```

### 6.2 代表数据集

| 数据集 | 规模 | 用途 |
|--------|------|------|
| Ego4D | 3000+ h | 语义、手物交互 |
| Ego-Exo4D | 更大 | ego-exo 对齐 |
| EgoScale | 20K h | scaling law |

### 6.3 关键误区

> **Ego 视频不能单独当 VLA motor BC 主粮**——缺 executable action 标签；属于 **Tier 1 prior**，需与 Tier 2/3 机器人数据配合。

**笔记**：[Ego视频数采与对齐](../paper-note/Data-Pipeline/Ego视频数采与对齐.md)

## 七、范式 E · 仿真与合成

### 7.1 类型

| 类型 | 代表 | 作用 |
|------|------|------|
| **Benchmark 仿真** | LIBERO, CALVIN | 评测 + 小样本 pipeline 验证 |
| **大规模厨房/家居** | RoboCasa | π 系列 pretrain bootstrap |
| **合成扩增** | MimicGen | 少量 seed → 大量变体 |
| **传统 IL 仿真** | robomimic | Diffusion Policy 基线 |

### 7.2 定位

```text
仿真 ≠ 替代真机 Tier 3
仿真 = 低成本覆盖 + 算法调试 + 与真机混训（sim2real gap 需正视）
```

| 范式 | Fidelity | Scale | 成本 | 有 action 标签 | 典型 VLA 用途 |
|------|----------|-------|------|----------------|---------------|
| **A Teleop** | ★★★★★ | ★★ | 高 | ✅ 精确 | 微调 Tier 3 |

## 九、从数采到训练的管线

```text
1. Protocol 设计（相机、频率、任务、语言模板）
2. 采集（A–E 之一或组合）
3. QA（丢帧、静止段、失败 ep、时间戳）
4. 标注（语言、成功/失败、segment）
5. Canonical IR（统一步/观测/action 语义）
6. 导出 RLDS（预训练）或 LeRobot（微调）
7. 混合 Recipe + Collator → VLA 训练
```

**总方案**：[数采到VLA训练-数据管线整体方案](../数采到VLA训练-数据管线整体方案.md)

## 十、选型指南

| 你的情况 | 推荐 |
|----------|------|
| 首次跑通 VLA 微调 | ALOHA sim Teleop → LeRobot |
| 低成本扩数据、无真机 | UMI 路线 |
| In-the-wild 泛化 | DROID 类 Teleop + 多场景 |
| 双臂精细 | ALOHA 真机 Teleop |
| 只研究语义/表征 | Ego4D + 对齐论文 |
| 算法 ablation | LIBERO / robomimic |
| 工业数采平台 | MCAP → IR 双轨导出 |

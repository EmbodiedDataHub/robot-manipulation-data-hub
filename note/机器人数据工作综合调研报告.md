## 一、Executive Summary

机器人操作正在经历与 NLP/CV 类似的 **「数据基础设施优先」** 转变。2026 年多篇综述（尤其 arXiv:2604.23001）明确指出：**VLA 的下一波突破更依赖 Data Engine 与 Evaluation Protocol 的 co-design，而非单纯堆模型参数**。

对「做数据工作」的同学，核心结论是：

| 维度 | 2023 认知 | 2026 认知 |
|------|-----------|-----------|
| 瓶颈 | 机器人 demo 太少 | demo **质量、标注、多样性、失败覆盖**不足 |
| 解法 | 多采 teleop | **Data Engine**（视频→轨迹、仿真生成、手持野外、fleet 级 teleop）+ **严格 QA** |
| 规模 | 单 lab 百小时 | 跨 embodiment **万~十万小时**（OXE、RDT2、EgoScale、Qwen-RobotManip） |
| 格式 | 各 lab 自定义 HDF5 | **RLDS**（预训练）+ **LeRobot**（HF 生态）双轨并行 |
| 评估 | LIBERO success rate | 扰动下成功率暴跌 → 需要 **LIBERO-Plus/PRO** 等诊断 benchmark |

**本地已下载 9 篇综述 PDF**，路径：[`paper/Surveys/`](../paper/论文索引.md#external-surveys)（详见[论文索引](../paper/论文索引.md) · [资源索引](./resources/links/资源索引.md)）。

## 二、数据工作的全景地图

机器人数据工作不是「录完丢进训练」，而是一条完整产业链：

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Layer 5: 训练混合 & Foundation Model                                    │
│  OXE 预训练 · RDT/Qwen/π0 混合 recipe · 负迁移诊断 · Data Pyramid        │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↑
┌─────────────────────────────────────────────────────────────────────────┐
│  Layer 4: 数据管理 & 交付                                                │
│  版本化 · Provenance · Robo-DM 压缩 · LeRobot/RLDS 导出 · Datasheet      │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↑
┌─────────────────────────────────────────────────────────────────────────┐
│  Layer 3: 清洗 & 增强                                                    │
│  QA 过滤(去掉底部20-40%) · 语言重标注 · MimicGen 合成 · 失败轨迹保留       │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↑
┌─────────────────────────────────────────────────────────────────────────┐
│  Layer 2: 采集                                                           │
│  Teleop · 手持(UMI/Koala) · VR 人形 · Ego 视频 · 仿真 · Internet 视频    │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↑
┌─────────────────────────────────────────────────────────────────────────┐
│  Layer 1: 协议设计                                                       │
│  任务定义 · 传感器配置 · Action 表示 · 成功/失败标准 · 多样性 protocol    │
└─────────────────────────────────────────────────────────────────────────┘
```

**工程侧关键数字**（来自 SVRC / Truelabel / Claru 等实践指南）：

- 操作员成本约 **$30–50/小时**；1000 episodes 直接成本 **$5k–15k**
- 无 QA 时 **30–40%** episode 不可用
- **去掉质量最差 20%** 往往比 **多加 50% 数据** 更能提升策略成功率
- 多模态同步目标：**< 10ms** 时间戳偏差
- π0 / 工业 VLA 常见控制频率：**50 Hz**

## 三、五条数据采集范式

结合原报告四条主线 + 2026 行业调研，可归纳为 **五条互补范式**：

| 范式 | 代表工作 | 是否需要机器人 | Fidelity | Scale | 力/触觉 | 典型 Action |
|------|----------|----------------|----------|-------|---------|-------------|
| **A. 实验室 Teleop** | ALOHA, GELLO, Mobile ALOHA | ✅ | ⭐⭐⭐⭐⭐ | 小 | 可选 | 关节角 / EEF |
| **B. 手持野外 Interface** | UMI, FastUMI, DexWild, Koala(RAI) | ❌ | ⭐⭐⭐⭐ | 中→大 | Koala 有 | 相对 EEF 轨迹 |
| **C. VR / 人形 Teleop** | Open-TeleVision, OPEN TEACH, TWIST2 | ✅ | ⭐⭐⭐⭐ | 中 | 通常无 | whole-body |
| **D. 人类 Ego / Internet 视频** | EgoScale, Phantom, EgoVLA, VideoDex | ❌ | ⭐⭐ | 极大 | 无 | 需 retarget / latent |
| **E. 仿真 / 合成引擎** | MimicGen, RoboGen, InternData-A1 | ❌(虚拟) | ⭐⭐⭐ | 极大 | 可模拟 | 与 sim 一致 |

### 范式选型决策树

```
你的任务是否需要精确力控 / 接触-rich？
├─ 是 → A (teleop+力传感) 或 B+力传感 co-design (Koala)
└─ 否 → 是否需要 cross-embodiment 野外泛化？
         ├─ 是 → B (UMI 系) 或 D→B 混合
         └─ 否 → 是否有预算搭 lab teleop？
                  ├─ 是 → A
                  └─ 否 → D (ego) + 少量 B 校准
```

### 各范式 2026 新认知

**A. Teleop** — 仍是 **最高 fidelity** 金标准，但成本高、难 scale。Claru 指南强调：操作员培训（practice 20–50 episodes）、3–5 人轮换、实时监控 success rate / episode duration / 任务覆盖度。

**B. 手持** — UMI 证明「无机器人采数据」可行；RAI Koala 进一步论证 **采集器与执行器 co-design**（同一套手指几何 + 传感器）可缩小 embodiment gap，并保留 **力信号**——这是纯视觉 UMI 做不到的。

**C. VR 人形** — Open-TeleVision 的 **主动立体头显** 解决遮挡；适合 long-horizon 精细任务和 cross-site 远程采集（60Hz）。

**D. Ego 视频** — 规模天花板最高（EgoScale 20k+ 小时），但 **action-free、遮挡、retarget** 三大难题仍在。2026 文献共识：人类视频是 **预训练 prior**，不能替代 robot-aligned 数据做 final calibration。

**E. 合成** — MimicGen（200→50K）是「种子 demo 扩增」标杆；InternData-A1 等探索 **纯合成匹配真实**；但 failure mode 覆盖不足，需 VLM critic 过滤。

## 四、数据基础设施：格式 / 管线 / 引擎

### 4.1 存储格式怎么选

| 格式 | 何时用 | 优点 | 缺点 |
|------|--------|------|------|
| **RLDS** | OXE 预训练、跨 lab 贡献 | 统一 episode/step schema、TFDS 生态 | TensorFlow 倾向、转换成本高 |
| **LeRobot** | HF 开源栈、SmolVLA/ACT | Parquet+MP4、Hub 共享、PyTorch 友好 | 较新、部分老代码不兼容 |
| **HDF5** | ACT/Diffusion Policy 复现 | 成熟、随机访问 | 跨项目 schema 不统一 |
| **MCAP** | ROS2 录制 | 实时、可索引 | 需后处理转训练格式 |
| **Robo-DM/EBML** | 超大规模 | 70× 压缩、50× 解码加速 | 生态较新 |

**实践建议**：采集阶段 MCAP/ROS2 或原生 LeRobot recorder → QA 后导出 **LeRobot（微调）** + **RLDS（若贡献 OXE）** 双份。

### 4.2 生产级管线六阶段

综合 SVRC + Humaid 博客：

1. **Protocol Design** — 任务、传感器、episode 结构、成功标准、**多样性 protocol**（光照/背景/干扰物变化）
2. **Synchronized Recording** — 硬件时间戳、NTP/chrony、单 recorder 节点
3. **Quality Filtering** — 自动：抖动检测、blur、timestamp drift、joint limit；人工：10–20% spot check
4. **Annotation** — 语言指令（避免模板化！）、segment、object id、grasp type、**失败标签**
5. **Format Conversion & Versioning** — Git tag / DVC；**Datasheet for Datasets**
6. **Training Integration** — 与 lerobot-train / Octo loader 对接验证

### 4.3 Data Engine 三分法（arXiv:2604.23001）

| 类型 | 机制 | 代表 | 数据工作关注点 |
|------|------|------|----------------|
| **Video-to-Data** | 人类视频 → 轨迹/affordance/latent action | EgoVLA, Phantom, RoboWheel | 对齐 pipeline、action 标注质量 |
| **Hardware-Assisted** | Teleop / 手持 / mocap | UMI, ALOHA, DROID, DexCap | 接口设计、latency matching |
| **Generative** | 仿真 + LLM 任务生成 + 扩增 | MimicGen, RoboGen, SynGrasp-1B | 物理 grounding、sim2real 验证 |

## 五、2024–2026 关键综述解读

### ⭐ 必读：VLA Datasets, Benchmarks, and Data Engines (2604.23001)

**为什么读**：第一篇 **纯数据-centric** 的 VLA 综述，直接对应「做数据」岗位。

**核心框架**：
- **Datasets**：Real vs Synthetic；按 embodiment diversity、modality、action space 三轴分类
- **Benchmarks**：Task complexity × Environment structure 二维地图；指出 LIBERO 等 short-horizon 基准的局限
- **Data Engines**：上述三分法

**数据集选型指南（原文 §3 总结）**：

| 目标 | 推荐数据集 |
|------|-----------|
| 跨 embodiment 预训练 | Open X-Embodiment |
| 单平台微调 | RT-1, BridgeData V2, DROID |
| 环境鲁棒性 | DROID（in-the-wild 多样性） |
| 接触-rich | RH20T（力/触觉/音频） |
| 人类语义 prior | RT-2 co-train, Ego4D |
| 仿真 bootstrap | RoboCasa, MimicGen, SynGrasp-1B |

**配套资源**：https://github.com/ziyaow1010/vla-datasets-benchmarks

---

### Open X-Embodiment (2310.08864)

- **1M+ episodes, 22 robots, 60 子数据集, 34 机构**
- **RLDS** 统一格式 → 异构 action space / 相机数量均可容纳
- **关键经验**：RT-1-X 跨 robot 迁移成功率 **+50%**；但 2026 后续工作警告 **盲目混合可能负迁移**
- **数据工作启示**：贡献 OXE 需写 TFDS DatasetBuilder，action 常归一化到 7-DoF EEF delta

---

### Robo-DM (2505.15558)

- Berkeley 开源 **EBML** 格式 + 云原生工具链
- vs RLDS：**70× 有损 / 3.5× 无损** 压缩；vs LeRobot：**50×** 顺序解码加速
- **数据工作启示**：万小时级数据集必须同时优化 **存储、传输、加载** 三环

## 六、行业共识：数据瓶颈已从「量」转向「质」

Label Studio 对 **1228 篇 VLA 论文** 的 meta 分析（2023.02–2026.06）提炼出与数据工作直接相关的结论：

### 6.1 旧问题：量不够 → 新问题：质不行

- 早期：**「robot trajectory 太贵采」**（150+ 篇论文的共同开头）
- 现在：**「 naive scaling 失效」**
  - 异构数据混合 → **负迁移**（arXiv:2602.09722）
  - 5% curated coreset → 恢复 **85–90%** 全量性能（arXiv:2511.16233）
  - **SEVO**：采集时变化光照/背景/干扰物 → 泛化最关键；纯 in-distribution 数据 → 近零 OOD 迁移

### 6.2 三个被忽视的 Data Gap

| Gap | 现象 | 低成本修复 |
|-----|------|-----------|
| **语言 impoverished** | 指令模板化，VLA 忽略语言输入 | **Counterfactual relabeling**（CAST +27pp）；LangGap 0→90% |
| **失败数据被丢弃** | 只训 success demo | 保留 slip/collision 轨迹；RoboFAC / FailSafe 等新数据集 |
| **模态缺失** | 1228 篇中 tactile 几乎空白 | RH20T、ForceVLA、HapTile 等仍处「旗舰级稀缺」 |

### 6.3 评估危机 = 数据问题的镜像

- LIBERO 在 300+ 论文中使用，但 LIBERO-Plus/PRO 显示扰动下 **95% → <30% / 0%**
- 小样本 real eval（≤25 rollouts）无置信区间 → 数据改进无法可靠度量
- **数据工作者的责任**：建立与部署场景匹配的 eval protocol，而非刷 sim benchmark

## 九、开放问题与 2026 趋势

### 9.1 开放问题

1. **Unified Action Space 的标准** — RDT / Qwen / OXE 各有一套，跨 FM 互换仍困难
2. **Human video 的最小 robot 校准量** — Phantom/EgoZero 说 zero-shot，工业界仍要 hundreds hrs teleop
3. **Failure-centric training** — 数据有了（RoboFAC 等），训练 recipe 未收敛
4. **Tactile / Force at scale** — 传感器便宜但 **aligned multimodal dataset** 几乎空白
5. **Benchmark 可信度** — 需要 real-world eval + 扰动协议成为标配
6. **Data security** — 0.31% poison → 99% backdoor success（VLA 特有攻击面）

### 9.2 2026 趋势预测

| 趋势 | 信号 |
|------|------|
| **Data Engine as a Product** | SVRC、Claru、EmbodiFlow 等商业化管线 |
| **Fleet-scale 手持采集** | RDT2: 100 设备 × 100 家庭 |
| **Synthetic-only 预训练** | InternData-A1 |
| **Language relabeling 工具链** | CAST, LangGap 证明 ROI 极高 |
| **LeRobot 成为默认交换格式** | HF 生态 + 多家 FM 支持 |
| **Co-design 手持 + 力传感** | RAI Koala、DexUMI |
| **Diagnostic benchmarks** | LIBERO-Plus/PRO/X 系列 |
